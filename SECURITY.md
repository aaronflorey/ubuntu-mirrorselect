# Security Policy

## Supported Versions

Security fixes are applied to the latest release and the current `main` branch.

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Use GitHub's private vulnerability reporting for this repository when it is available. If private reporting is not enabled, open a regular issue only to request a private contact path and do not include sensitive details, exploit steps, or proof-of-concept payloads.

Please include:

- A clear description of the issue.
- The affected version or commit.
- Reproduction steps or a minimal proof of concept.
- Any suggested remediation or mitigation.

We will acknowledge valid reports as quickly as practical and coordinate disclosure once a fix is available.
